# GPT-image-2 刚上线，我认真试了试：普通人到底该怎么用，Plus 和 Pro 值不值得开，Codex 是否支持

先说结论如果需要生图需求，Plus一定值得开通....不信你往下看！

昨天 OpenAI 把新一代图片能力放出来之后，我用了下，网上一片哀嚎，出的图堪比设计师，最直接的感受不是“AI 又会画图了”，而是它终于开始像一个能真正干活的工具了。

以前很多人提到 AI 生图，脑子里还是那种偏试玩的印象：做头像、做壁纸、偶尔整点夸张海报，热闹归热闹，真到了写文章、做页面、做产品图、做封面的时候，反而不太敢把它当正经工具用。可这次不一样，新模型已经不只是把图画得更好看一点，而是把“生成—修改—继续迭代”这条链路补得更完整了。

OpenAI 在 2026 年 4 月 21 日把它作为 ImageGen 2.0 带进了 ChatGPT，而且是面向所有 ChatGPT 套餐开放。

如果从开发者文档去看，这个能力对应的模型名就是 gpt-image-2。官方现在把它列成最新的图像生成模型，支持高质量生成和编辑，也支持图像输入，不只是“看图说话”，而是真正能在文字和图片之间来回工作。

一个很明显的变化是：它更像在接需求，而不是在猜你想要什么。这个差别其实很大。以前很多生图工具表面上看也能出图，但你越有明确需求，它反而越容易跑偏；这次 GPT-image-2 给我的感觉是，只要你说得具体，它真的会尽量往你要的方向靠。OpenAI 这次公开展示的重点里，也专门强调了更好的文字渲染、多语言表现、复杂版式、风格控制和更强的真实世界知识。

# 先把最容易搞混的地方说清楚

现在很多人会在不同地方看到不同名字，这很正常。现在plus用户打开ChatGPT的图片模型就是最新的gpt-image-2。

在 ChatGPT 产品里，你更常看到的是 ImageGen 2.0 或 ChatGPT Images 2.0。在 OpenAI 的开发者文档里，对应的是 gpt-image-2。不用把这两个名字理解成两套东西，本质上它们说的是同一代新的图像能力，只是一个是产品侧叫法，一个是模型侧叫法。OpenAI 的 ChatGPT 更新日志写的是 ImageGen 2.0，帮助中心写的是 ChatGPT Images 2.0，开发者文档则直接写 gpt-image-2。

这个区别其实挺重要。因为如果你只是普通用户，最顺手的入口还是 ChatGPT 官网和手机 App；如果你是开发者，或者你做的网站、工具本身就要接图像生成，那才会走到 API 那一侧。两边能力有交集，但使用方式完全不是一回事。

# 现在到底能在哪些地方用

最省心的地方，肯定还是 ChatGPT 官网和手机 App直接让生成图片就可以使用。

OpenAI 的帮助页已经写得很清楚：ChatGPT Images 可以直接在对话里创建新图，也可以编辑已有图片；现在支持 Web 和 iOS/Android，而且图片会自动保存在 Images 入口里，后面可以继续回看、复用和修改。你甚至不需要专门跳去别的页面，只要在聊天里直接说需求，或者从侧边栏进入 Images 就能开始。

这次我觉得最值钱的，不是“从零画一张图”，而是“画完还能继续改”。官方现在给了两种编辑方式：一种是选中图片某个区域，再告诉它怎么改；另一种是不圈区域，直接在对话里描述修改内容。它还支持透明背景、任意比例、局部加细节、替换元素，已经很接近真实工作里常见的改图流程了。

除此之外，GPTs 这条线其实也很实用。官方帮助页明确写了，只要一个 GPT 开启了 Image Generation 能力，它就能使用新的图片生成模型。这个点很多人容易忽略，但对经常做固定风格内容的人特别有用。比如你可以专门做一个“公众号封面 GPT”“电商主图 GPT”“产品海报 GPT”，把风格要求、口吻、比例偏好提前放进去，后面就不用每次从头讲一遍。

如果你是开发者，或者你想把图片生成接进自己的产品里，那么就会用到 API。OpenAI 现在在文档里把 gpt-image-2 定义成最新的图像生成模型，支持通过 Image API 直接生成或编辑图片，也支持通过 Responses API 把图片生成放进更长的多轮对话和工作流里。简单说，前者适合“一次请求，一张或几张图”，后者适合“边聊边改，连续迭代”。

还有你最关心的 Codex。现在这块不是不能讲，而是要讲得干净。对开发者来说，Codex 最大的意义不是“它也能画图”，而是代码和图片终于开始进同一个工作流了。OpenAI 的 Codex CLI 功能页已经写明，内置图片生成功能使用的是 gpt-image-2，并且这部分会计入 Codex 的一般使用额度；IDE 端也支持直接在编辑器里生成或编辑图片，还能用 $imagegen 明确调用这项能力。换句话说，做前端页面、UI 资产、占位插图、mockup、游戏素材这些事，现在确实已经可以在 Codex 体系里顺手完成，而不是一定要在代码工具和设计工具之间来回切。

# 普通用户最该先学会的，不是提示词技巧，而是提需求的方式

我这几天试下来，最大的感受其实特别朴素：别把自己逼成什么“提示词工程师”，先学会像跟设计同事提需求一样说话。

很多人第一次上手，容易直接来一句“帮我画个高级感海报”“来一张赛博朋克封面”“做个有电影感的图”。这类话并不是完全没用，但它太空了。什么叫高级感？是奢侈品广告的高级，还是杂志摄影的高级，还是极简产品图的高级？什么叫电影感？是冷调都市夜景，还是暖色室内静物，还是胶片颗粒和逆光？你不说清楚，它只能靠猜。

我现在更推荐一种很简单的说法：把需求拆成用途、主体、场景、风格、构图、限制这六件事。

比如说下面这种表达方式其实一点都不神秘，但它会比一堆堆砌关键词更有效。

“给我做个知乎封面”，而是说：“帮我生成一张适合知乎文章封面的横版图片，主题是一个人在夜里安静地对着电脑整理思路，桌上有咖啡、笔记本和台灯，整体偏真实摄影感，色调克制一点，不要太赛博朋克，左上角留标题区，16:9。

OpenAI 这次给到的官方资料里，也明显把“可控性”和“复杂指令遵循”放在很前面的位置：包括高质量写实、风格迁移、复杂结构图、清晰文字、版式控制、人物一致性，以及更强的编辑表现。说白了，它已经不是那种只能喂几个风格词碰碰运气的阶段了。

# 真正好用的方式，不是一步到位，而是分三轮走

这一点我特别想单独说，因为它基本决定了你会不会觉得这东西“真好用”。

很多人用生图工具，总希望第一轮就直接出终稿。结果往往是：需求写得又长又满，出来的图却不是重点太散，就是信息太乱。其实现在这类模型最适合的方式，反而不是一步到位，而是分层推进。

# 第一轮，只定方向

先看气质对不对。是偏真实、偏插画、偏杂志、偏广告，还是偏海报；整体情绪是冷还是暖，是克制还是张扬。这一轮不要急着抠局部。

# 第二轮，再改构图

主体够不够突出，背景会不会太挤，标题区有没有留够，产品是不是站住了，人物是不是太抢。真正拿来发文、做封面、做页面，构图往往比“画得精不精”更影响结果。

# 第三轮，才去修细节

像光线、肤感、材质、物件数量、背景层次、手部自然度、局部精修，这些放到最后一轮，成功率会更高。

为什么我会这么建议？因为 ChatGPT Images 现在本身就支持反复编辑、局部编辑和重新设定比例。它的工作方式已经不是“一次出图，错了就重来”，而更像“先给你一版，再根据反馈往前修”。这其实非常像真实工作场景。

# 在 ChatGPT 里怎么用，效率最高

如果你第一次上手，我真不建议一开始就研究各种复杂模板。最省事的方式，就是把它当成一个会画图、而且听得懂人话的搭子。

你可以直接在聊天里说：“帮我生成一张图。”
也可以更进一步：“帮我生成一张适合公众号头图的横版图片，主题是凌晨还在写代码的人，桌上有咖啡和台灯，真实摄影感，蓝灰色调，右边留白，16:9。”

如果你已经有第一版了，下一步不要重新开新话题，直接接着改。比如：

  • 人物往右移一点
  • 背景别这么满，干净一点
  • 把桌面上的杂物去掉
  • 整体从冷调改成偏暖
  • 别让人物看镜头
  • 标题区再留大一点

这一类连续修改，往往比“从零重新来一张”更稳。因为模型已经理解了你这张图的主体和方向，后面的修改更像是在同一个语境里微调，而不是重新猜一次。官方现在也支持在编辑器里直接选中区域再修改，移动端同样可以做。

还有一个挺舒服的点是，图片会自动保存在 Images 里。很多时候你当下不用，过两天回头看，反而会发现某张草稿很适合别的内容。这个细节其实很实用，因为创作里最怕的就是每次都从零开始。

# 真正决定出图质量的，不是“华丽词汇”，而是信息密度

我现在越来越不信那种“万能提示词模板”了。真正管用的，不是你写得像不像广告文案，而是你有没有把关键信息说到位。

最有效的提示，一般都同时解决了这几件事：

  • 这张图是干什么用的
  • 画面里谁是主角
  • 它发生在什么场景
  • 希望它看起来像什么
  • 比例和留白怎么安排
  • 哪些东西不要出现

比如做产品海报，你说“帮我做个高级感护手霜海报”，远不如这样说有效：

帮我生成一张护手霜海报，主角是一支白色极简包装的护手霜，放在浅米色石材台面上，背景是柔和自然光下的浴室空间，整体像高端护肤品牌广告，干净、克制、偏真实摄影，竖版 4:5，产品位于画面中下部，上方留白给文案，不要出现多余产品，不要让背景太乱。

你就说这牛皮不嘛？？？？

这类写法没有任何玄学，它就是把一张图真正需要的信息讲清楚了。

OpenAI 在这次的提示指南里其实也给出了很一致的方向：gpt-image-2 更适合生产级视觉、可控创意工作流、文字清晰的图内排版、复杂结构图、多步编辑、精确风格控制和更强的真实世界表现。它的重点不是“越花哨越好”，而是“越清楚越好”。

# GPT-image-2 到底值不值得现在就开始用

如果你问我，这次 GPT-image-2 刚上线，到底值不值得现在就开始用，我的答案是：值，而且它已经不是“尝鲜玩具”了。

它最有价值的地方，不是让所有人一夜之间变成设计师，也不是保证每一张图都直接能商用。它真正改变的是一件更现实的事：以前你脑子里有一张模糊的图，要花很多时间去找素材、拼灵感、改来改去；现在你终于可以直接把那个想法说出来，再一点点把它推近你真正想要的样子。

这对写作者、产品人、运营、前端、独立开发者、小团队，甚至只是想把封面做得顺眼一点的普通人来说，都已经很够用了。

如果你只是想认真体验这次新图片能力，Plus 足够作为起点；如果你本来就高频使用 ChatGPT 和 Codex，Pro 才会把差距真正拉出来。至于技巧，我反而不觉得最重要的是研究什么“咒语”，而是把自己的需求说清楚，把它当成一个能来回沟通的视觉协作者，而不是一次性出神图的许愿机。

真要用一句话收尾，我会这么说：GPT-image-2 最像样的地方，不是它会画，而是它终于开始会配合。
